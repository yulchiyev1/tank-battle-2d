Detailed Description

Lightweight context container that provides access to core engine systems.

EngineContext is a non-owning structure used to pass references to core engine systems (such as rendering, input, audio, state management, and window management) throughout the engine and gameplay code.

Instead of relying on global variables or singletons, the engine passes EngineContext to functions such as GameState::Update(), Draw(), Object update loops, and resource loading.

All members are raw pointers and are owned by JinEngine. EngineContext does not manage the lifetime of these systems.

Typical usage includes:

  • Accessing RenderManager to submit draw calls
  • Accessing InputManager to query input state
  • Accessing StateManager to change game states
  • Accessing WindowManager for resolution or viewport information
  • Accessing SoundManager for audio playback
Note
All pointers are expected to be valid during runtime after engine initialization.
Warning
EngineContext does not perform null checks internally. Users must ensure that the engine is properly initialized before accessing its members.
