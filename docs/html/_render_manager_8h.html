◆ RenderMap

using RenderMap = std::array<std::map<int, ShaderMap>, RenderLayerManager::MAX_LAYERS>

Stores the full frame's render organization by render layer and depth bucket.

RenderMap is indexed by render layer first. Each layer stores a map keyed by quantized depth. Each depth entry then contains a ShaderMap.

This means the effective ordering is:

  • render layer
  • quantized depth
  • shader
  • instance batch key
  • object list

RenderManager::BuildRenderMap() fills this structure, and RenderManager::FlushDrawCommands() consumes and clears it.

◆ ShaderMap

using ShaderMap = std::unordered_map<Shader*, std::map<InstanceBatchKey, std::vector<Object*>>>

Groups render batches first by shader, then by batch key, then by object list.

The engine uses this structure while building the frame's render map. For a given depth bucket inside a given render layer, objects are grouped by:

  • Shader*
  • InstanceBatchKey
  • std::vector<Object*>

This grouping lets RenderManager later decide whether to draw objects one by one or through the instancing path.
