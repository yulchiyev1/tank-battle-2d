Wraps an already existing OpenGL texture object.

This constructor does not allocate GPU storage or upload any pixel data. It simply stores an existing OpenGL texture ID together with metadata such as width, height, and channel count so the engine can treat the external texture like a regular Texture resource.

This is mainly used by internal engine systems that create textures through other OpenGL paths, such as render targets or externally managed resources.

Parameters
id_Existing OpenGL texture object ID.
width_Width of the texture in pixels.
height_Height of the texture in pixels.
channels_Number of color channels represented by the texture data.
Note
Since this object takes ownership of the texture ID at the wrapper level, the destructor will delete the OpenGL texture if the stored ID is non-zero.

◆ Texture() [2/3]

Texture::Texture ( const FilePath & path,
const TextureSettings & settings = {} )

Loads an image file from disk and creates a GPU texture from it.

The implementation uses stb_image to load the file contents. Vertical flip on load is enabled before reading, so image data is imported in a form that matches the engine's texture coordinate convention.

After loading succeeds, the constructor forwards the pixel buffer and the requested TextureSettings to GenerateTexture(), which allocates immutable texture storage, uploads the base image, applies filter and wrap state, and optionally generates mipmaps.

If loading fails, the texture remains with an invalid or empty state and an error is logged.

Parameters
pathPath to the image file to load.
settingsTexture creation settings such as filtering, wrapping, and mipmap generation.
Note
The loaded CPU-side image buffer is released immediately after the GPU texture has been created.

Creates a GPU texture directly from raw pixel data in memory.

This constructor is used when image bytes are already available in memory, such as asynchronously loaded textures, generated images, font atlases, or procedural content. The provided width, height, and channel count are stored first, then GenerateTexture() creates the OpenGL texture and uploads the supplied data.

Supported channel counts in the implementation are 1, 3, and 4, which map to red, RGB, and RGBA texture formats respectively.

Parameters
dataPointer to raw pixel data to upload. Passing nullptr creates an allocated texture with cleared contents.
width_Width of the texture in pixels.
height_Height of the texture in pixels.
channels_Number of channels in the source pixel data.
settingsTexture creation settings such as filtering, wrapping, and mipmap generation.

Binds this texture as an image unit for shader image access.

This path is intended for compute shaders or other shader stages that use image load/store operations rather than sampler-based reads. The access mode and image format are converted to the corresponding OpenGL enums and bound through glBindImageTexture().

Parameters

formatDeclared image format exposed to the shader.

Recreates the texture from raw pixel data and replaces the current GPU object.

If this Texture already owns a valid OpenGL texture object, the old object is deleted first. The stored metadata is then updated and GenerateTexture() is called to allocate and upload the new texture contents.

This is used when the engine wants to refresh an existing registered texture entry with new image data while preserving the same resource handle in the resource map.

Parameters
dataPointer to the new pixel buffer.
width_Width of the new texture in pixels.
height_Height of the new texture in pixels.
channels_Number of channels in the new pixel data.
settingsTexture creation settings for the recreated texture.

◆ GenerateTexture()

void Texture::GenerateTexture ( const unsigned char * data,
const TextureSettings & settings )
private

Allocates GPU storage and uploads pixel data according to the given settings.

This internal helper chooses the OpenGL internal format and upload format from the stored channel count, creates the texture object, allocates immutable storage, uploads the base image if data is present, applies sampler parameters, and optionally generates mipmaps.

When data is nullptr, the implementation still creates the texture and clears its contents so the texture can be used as an empty resource or destination surface.

Parameters
dataPixel buffer to upload. May be nullptr.
settingsTexture creation settings applied to the new OpenGL texture.
Note
The implementation temporarily forces GL_UNPACK_ALIGNMENT to 1 so tightly packed image data uploads correctly regardless of row alignment.
