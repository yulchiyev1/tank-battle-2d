Transform2D::Transform2D ( )
inline

Constructs a default 2D transform.

Initializes the transform with:

  • position = (0, 0)
  • rotation = 0
  • scale = (1, 1)
  • matrix = identity
  • isChanged = true
  • depth = 0

The matrix is not rebuilt immediately here. Instead, it is marked dirty and lazily recalculated the next time GetMatrix() is called.

Note
The depth value is stored separately from the transform matrix and is used for render ordering rather than geometric transformation.

◆ GetMatrix()

glm::mat4 & Transform2D::GetMatrix ( )
nodiscard

Returns the cached transform matrix.

Rebuilds the matrix only when the transform has changed. In the current implementation, the matrix is composed as:

position translation * offset translation * rotation * scale

After rebuilding, the dirty flag is cleared and the cached matrix is reused until another transform-changing function marks it dirty again.

Returns
Reference to the internally cached 4x4 transform matrix.
Note
The returned reference remains valid as long as this Transform2D instance exists. The depth value is not included in this matrix.

void Transform2D::SetDepth ( float dpth)
inline

Sets the render depth value.

Stores the depth used for render ordering. This value does not mark the transform matrix as dirty because it is not included in the matrix built by GetMatrix().

In the current rendering flow, depth is read separately and quantized for render-map sorting.

◆ SetRotation()

void Transform2D::SetRotation ( float rot)
inline

Sets the rotation angle of the transform.

Replaces the current rotation value and marks the matrix as dirty. The actual matrix rebuild happens lazily in GetMatrix().

The rotation is applied around the Z axis in the generated 4x4 matrix.

Parameters
rotNew rotation angle.
Note
This implementation passes the stored value directly into glm::rotate(). In the current codebase, that means the unit must match what the rest of your engine expects when building transforms.
