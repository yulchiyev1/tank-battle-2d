\doxysubsection{Detailed Description}
Stores the bidirectional mapping between render layer names and fixed layer IDs. 

\doxylink{class_render_layer_manager}{Render\+Layer\+Manager} is a small internal registry used by \doxylink{class_render_manager}{Render\+Manager} to manage the engine\textquotesingle{}s named render layers.

Each layer is identified by\+:
\begin{DoxyItemize}
\item a string tag used by user-\/facing code and objects
\item a compact uint8\+\_\+t layer ID used internally for ordering and batching
\end{DoxyItemize}

The manager maintains two synchronized lookup containers\+:
\begin{DoxyItemize}
\item name\+To\+ID \+: resolves a layer name to its numeric ID
\item id\+To\+Name \+: resolves a numeric ID back to its layer name
\end{DoxyItemize}

The current design supports at most MAX\+\_\+\+LAYERS entries, and each layer ID can only be assigned once. Registration fails if\+:
\begin{DoxyItemize}
\item the name already exists
\item the requested layer ID is out of range
\item the requested layer ID is already occupied
\end{DoxyItemize}

In the current engine flow, \doxylink{class_render_manager}{Render\+Manager} owns this manager and uses it when\+:
\begin{DoxyItemize}
\item registering a new render layer by tag
\item unregistering a layer
\item resolving an object\textquotesingle{}s render layer tag during draw batching
\end{DoxyItemize}

\begin{DoxyNote}{Note}
This class is primarily an internal helper for \doxylink{class_render_manager}{Render\+Manager} rather than a high-\/level standalone system. 
\end{DoxyNote}

On failure, the function logs a warning or error and returns false.
