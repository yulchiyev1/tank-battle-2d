Stores the full frame\textquotesingle{}s render organization by render layer and depth bucket. 

\doxylink{_render_manager_8h_ac442dd962bc57bfa60125f977d6edea5}{Render\+Map} is indexed by render layer first. Each layer stores a map keyed by quantized depth. Each depth entry then contains a \doxylink{_render_manager_8h_a6dc798d95abd4b3fe7fe5051c9920759}{Shader\+Map}.

This means the effective ordering is\+:
\begin{DoxyItemize}
\item render layer
\item quantized depth
\item shader
\item instance batch key
\item object list
\end{DoxyItemize}

\doxylink{class_render_manager_a8e033cb685d2619f9b5dfec8a165e387}{Render\+Manager\+::\+Build\+Render\+Map()} fills this structure, and \doxylink{class_render_manager_a620231f7f4119dbf26ef416478578cde}{Render\+Manager\+::\+Flush\+Draw\+Commands()} consumes and clears it.

Groups render batches first by shader, then by batch key, then by object list. 

The engine uses this structure while building the frame\textquotesingle{}s render map. For a given depth bucket inside a given render layer, objects are grouped by\+:
\begin{DoxyItemize}
\item Shader\texorpdfstring{$\ast$}{*}
\item \doxylink{struct_instance_batch_key}{Instance\+Batch\+Key}
\item std\+::vector$<$\+Object\texorpdfstring{$\ast$}{*}$>$
\end{DoxyItemize}

This grouping lets \doxylink{class_render_manager}{Render\+Manager} later decide whether to draw objects one by one or through the instancing path.
