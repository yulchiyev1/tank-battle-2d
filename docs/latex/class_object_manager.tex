\doxysubsection{Detailed Description}
Owns, initializes, updates, renders, and removes runtime objects for the active game state. 

\doxylink{class_object_manager}{Object\+Manager} is the central container responsible for the lifetime flow of all state-\/owned \doxylink{class_object}{Object} instances. It manages several parallel views of the same object set\+:


\begin{DoxyItemize}
\item objects\+: The main owning container for active objects.
\item pending\+Objects\+: Newly added objects that have not yet completed the manager\textquotesingle{}s initialization flow.
\item object\+Map\+: A tag-\/to-\/object lookup table used for quick single-\/object queries.
\item raw\+Ptr\+Objects\+: A non-\/owning cache of raw pointers used by rendering, collision checks, and broad iteration.
\item broad\+Phase\+Grid\+: A spatial hash structure used to reduce the cost of collision pair generation.
\item collision\+Group\+Registry\+: A tag-\/to-\/bit registry used to build collision category and mask values.
\end{DoxyItemize}

In the current implementation, \doxylink{class_object_manager_ae1f29824763903213d6044d463742db4}{Add\+Object()} does not immediately place the object into the active owning list. Instead, it pushes the object into pending\+Objects and returns its raw pointer. The actual Init()/\+Late\+Init() flow and insertion into objects, object\+Map, and raw\+Ptr\+Objects happen later through \doxylink{class_object_manager_af5c6a21a87c2fa4204462cacbe07c439}{Add\+All\+Pending\+Objects()}.

\doxylink{class_object_manager_a00c892376f4874c2f033a69526e18111}{Update\+All()} is responsible for the main object tick flow. It updates alive objects, performs text-\/specific font/mesh refresh logic, advances attached sprite animators, synchronizes colliders with transform scale, removes dead objects, and finally integrates pending objects into the active list. \+:content\+Reference\mbox{[}oaicite\+:3\mbox{]}\{index=3\}

Rendering is delegated to \doxylink{class_render_manager}{Render\+Manager} through Submit(), using either all active raw pointers or filtered subsets. Collision checks are handled separately through \doxylink{class_object_manager_a1e9cd42ad2252270965253ddf57763de}{Check\+Collision()}, which first performs broad-\/phase pair generation with \doxylink{class_spatial_hash_grid}{Spatial\+Hash\+Grid} and then applies collision mask/category filtering before calling collider narrow-\/phase checks and \doxylink{class_object_a0230b42c88134134da384129e1a4d64d}{Object\+::\+On\+Collision()}.

\begin{DoxyNote}{Note}
This class is closely tied to \doxylink{class_game_state}{Game\+State} and acts as one of its internal core systems. 
\end{DoxyNote}

Initializes and activates all currently pending objects. 

This is the internal activation pipeline for newly queued objects.

The current implementation repeatedly drains pending\+Objects until it becomes empty. This allows object initialization code to enqueue additional objects without losing them.

For each drained object\+:
\begin{DoxyItemize}
\item Init(engine\+Context) is called first
\item if the object type is TEXT, Check\+Font\+Atlas\+And\+Mesh\+Update() is called immediately after Init()
\item the object is collected into a temporary activation list
\end{DoxyItemize}

After all pending waves are collected\+:
\begin{DoxyItemize}
\item Late\+Init(engine\+Context) is called
\item object\+Map\mbox{[}obj-\/\texorpdfstring{$>$}{>}Get\+Tag()\mbox{]} is updated
\item raw\+Ptr\+Objects receives the raw pointer
\item objects receives the unique\+\_\+ptr ownership
\end{DoxyItemize}

This delayed activation model keeps the main active list stable while new objects are being prepared.

Queues a new object for later initialization and activation. 

This function takes ownership of the supplied object and stores it in pending\+Objects rather than directly placing it into the active object list. A raw pointer to the same object is returned immediately so caller code can continue configuring it before or after the next manager update phase.

In the current implementation\+:
\begin{DoxyItemize}
\item the input object must not be null
\item the tag is assigned immediately through \doxylink{class_object_a5a8c0663fb3e668d16c8a045937a9c82}{Object\+::\+Set\+Tag()}
\item the object is pushed into pending\+Objects
\item actual Init()/\+Late\+Init() and active-\/list insertion happen later in \doxylink{class_object_manager_af5c6a21a87c2fa4204462cacbe07c439}{Add\+All\+Pending\+Objects()}
\end{DoxyItemize}

Unlike some earlier versions of the system, raw\+Ptr\+Objects and object\+Map are not updated here. That work is deferred until the pending activation step.

void Object\+Manager\+::\+Check\+Collision (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})}



Runs the engine\textquotesingle{}s collision detection flow for all active collidable objects. 

The current collision process works as follows\+:


\begin{DoxyEnumerate}
\item Create a checked\+Pairs set to avoid duplicate pair processing.
\item Clear the broad\+Phase\+Grid.
\item Insert every object that has a non-\/null collider into the spatial grid.
\item Ask the grid to generate candidate pairs.
\item For each pair\+:
\begin{DoxyItemize}
\item reject it if collision mask/category bits do not match
\item normalize pointer order for stable pair-\/key generation
\item skip it if already processed
\item run collider narrow-\/phase \doxylink{class_object_manager_a1e9cd42ad2252270965253ddf57763de}{Check\+Collision()}
\item if colliding, call On\+Collision() on both objects
\end{DoxyItemize}
\end{DoxyEnumerate}

The broad-\/phase grid itself uses a spatial hash and treats very large objects specially so they can still be paired against other objects without exploding per-\/cell insertion cost.

void Object\+Manager\+::\+Update\+All (\begin{DoxyParamCaption}\item[{float}]{dt}{, }\item[{const \mbox{\hyperlink{struct_engine_context}{Engine\+Context}} \&}]{engine\+Context}{}\end{DoxyParamCaption})}



Updates all active objects, removes dead ones, and activates newly queued ones. 

This is the main per-\/frame management function for runtime objects.

For each active object in objects\+:
\begin{DoxyItemize}
\item only alive objects are processed
\item if the object type is TEXT, Check\+Font\+Atlas\+And\+Mesh\+Update() is called first
\item Update(dt, engine\+Context) is called
\item if a \doxylink{class_sprite_animator}{Sprite\+Animator} is attached, it is advanced
\item if a \doxylink{class_collider}{Collider} is attached, Sync\+With\+Transform\+Scale() is called
\end{DoxyItemize}

After the main loop\+:
\begin{DoxyItemize}
\item Erase\+Dead\+Objects(engine\+Context) removes dead objects safely
\item Add\+All\+Pending\+Objects(engine\+Context) initializes and activates any newly queued objects
\end{DoxyItemize}

This means object creation and object destruction are both resolved through controlled post-\/update stages rather than during the middle of the main loop. \+:content\+Reference\mbox{[}oaicite\+:7\mbox{]}\{index=7\}
