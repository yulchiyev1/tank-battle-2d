\doxysubsection{Detailed Description}
Base interface for all collider shapes used by the engine. 

\doxylink{class_collider}{Collider} provides the common interface required by the collision system. Each collider is attached to an owning \doxylink{class_object}{Object} and may optionally follow the owner\textquotesingle{}s transform scale.

The class exposes point collision checks publicly, while pairwise collision checks, bounding radius queries, transform synchronization, and debug drawing are kept protected because they are intended to be driven by engine systems such as \doxylink{class_object_manager}{Object\+Manager} and \doxylink{class_spatial_hash_grid}{Spatial\+Hash\+Grid}.

Collision dispatch is implemented using double dispatch. The generic \doxylink{class_collider_a41660505ae4b413d3d4b004162857426}{Check\+Collision()} entry point forwards the test to the other collider, which then resolves the exact shape-\/vs-\/shape routine through \doxylink{class_collider_a6b1071065ccb7ddfc1ff8340a89246d7}{Dispatch\+Against()}.

\begin{DoxyNote}{Note}
\doxylink{class_collider}{Collider} does not own the \doxylink{class_object}{Object} pointer. The owner must outlive the collider.

When use\+Transform\+Scale is enabled, the collider shape is updated from the owner\textquotesingle{}s world scale during engine update. 
\end{DoxyNote}

When enabled, the collider shape is recalculated from the owner\textquotesingle{}s world scale through \doxylink{class_collider_a188380ff8ece20474e86a662f995887e}{Sync\+With\+Transform\+Scale()}. For \doxylink{class_circle_collider}{Circle\+Collider}, the largest absolute scale axis is used. For \doxylink{class_a_a_b_b_collider}{AABBCollider}, each axis is scaled independently. \+:content\+Reference\mbox{[}oaicite\+:1\mbox{]}\{index=1\}
