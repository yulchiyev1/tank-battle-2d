\doxysubsection{Detailed Description}
Manages sound resource registration, runtime playback instances, and global sound control for the engine. 

\doxylink{class_sound_manager}{Sound\+Manager} is the engine-\/level audio system built on top of miniaudio. It stores lightweight sound metadata through \doxylink{class_sound_manager_a8e52926369f4ff23599507c5e7cf7246}{Load\+Sound()}, and creates an actual playback instance only when \doxylink{class_sound_manager_acc7074efb4e8b5c71e85f556769d9b17}{Play()} is called.

Each \doxylink{class_sound_manager_acc7074efb4e8b5c71e85f556769d9b17}{Play()} call creates a new runtime sound instance, assigns it a unique \doxylink{_sound_manager_8h_a7675718ce8055fa8280410c0d32ad4f3}{Sound\+Instance\+ID}, stores it in an internal instance map, and also groups it by tag so that sounds can be controlled either individually or in batches.

The manager supports\+:
\begin{DoxyItemize}
\item loading sound metadata by tag,
\item starting multiple simultaneous instances from the same sound tag,
\item per-\/instance, per-\/tag, or global volume control,
\item per-\/instance, per-\/tag, or global pause/resume/stop control,
\item automatic cleanup of finished non-\/looping sounds,
\item optional completion callbacks for one-\/shot playback.
\end{DoxyItemize}

Pause and resume are implemented by saving the current PCM cursor position, stopping playback, and later seeking back to the saved frame before restarting.

Finished non-\/looping sounds are not removed immediately at the exact moment playback ends. They are collected and released during \doxylink{class_sound_manager_aaf241621221cdbefeba78e8b6bc29240}{Update()}, which internally calls \doxylink{class_sound_manager_a4a77d91e2c8257d6c2d02ffffad671ff}{Cleanup()}. Because of that, the engine must call \doxylink{class_sound_manager_aaf241621221cdbefeba78e8b6bc29240}{Update()} regularly every frame for proper sound lifecycle management.

\begin{DoxyNote}{Note}
\doxylink{class_sound_manager_a8e52926369f4ff23599507c5e7cf7246}{Load\+Sound()} only registers file path and loop information. Actual miniaudio sound objects are created per playback instance inside \doxylink{class_sound_manager_acc7074efb4e8b5c71e85f556769d9b17}{Play()}.

This class is owned and driven by the engine. \doxylink{class_jin_engine}{Jin\+Engine} is declared as a friend so it can access initialization, update, and shutdown functions. 
\end{DoxyNote}

void Sound\+Manager\+::\+Cleanup (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [private]}}



Cleans up finished non-\/looping sound instances and executes pending completion callbacks. 

This function scans all tracked runtime instances and identifies sound instances that\+:
\begin{DoxyItemize}
\item are not currently playing,
\item are not looping,
\item and have reached the end of the sound stream.
\end{DoxyItemize}

For each such instance, \doxylink{class_sound_manager_a4a77d91e2c8257d6c2d02ffffad671ff}{Cleanup()}\+:
\begin{DoxyItemize}
\item queues its completion callback if one exists,
\item uninitializes the miniaudio sound object,
\item removes the instance pointer from its tag-\/based active channel list,
\item deletes the runtime object,
\item erases the ID from instance\+Map,
\item pushes the released ID into the reusable ID queue.
\end{DoxyItemize}

After that, it also performs an additional pass over active channel vectors to remove stale pointers that correspond to naturally finished non-\/looping instances.

Finally, it executes queued callbacks after internal cleanup preparation is complete.

\begin{DoxyNote}{Note}
Looping sounds are intentionally skipped here and remain active until explicitly stopped. 
\end{DoxyNote}

Plays a registered sound using its tag and registers a callback for one-\/shot completion. 

This overload behaves the same as the simpler \doxylink{class_sound_manager_acc7074efb4e8b5c71e85f556769d9b17}{Play()} overload, but also stores an optional completion callback inside the runtime sound instance.

When a non-\/looping sound reaches the end of playback, \doxylink{class_sound_manager_a4a77d91e2c8257d6c2d02ffffad671ff}{Cleanup()} collects the callback and executes it after internal cleanup has been prepared. The callback receives the finished \doxylink{_sound_manager_8h_a7675718ce8055fa8280410c0d32ad4f3}{Sound\+Instance\+ID} and the original sound tag.

Looping sounds are not considered finished during normal playback, so their callback is not triggered automatically unless the sound is allowed to reach a natural end as a non-\/looping instance.

Updates the sound system for the current frame. 

This function currently forwards to \doxylink{class_sound_manager_a4a77d91e2c8257d6c2d02ffffad671ff}{Cleanup()}. It should be called regularly by the engine so that finished non-\/looping sounds are released, stale tracking entries are removed, reusable IDs are recycled, and pending completion callbacks are executed.

\begin{DoxyNote}{Note}
If \doxylink{class_sound_manager_aaf241621221cdbefeba78e8b6bc29240}{Update()} is not called regularly, finished one-\/shot sounds may remain in tracking containers longer than intended. 
\end{DoxyNote}
