\doxysubsection{Detailed Description}
Rendering material that stores a shader, uniform values, and texture bindings. 

\doxylink{class_material}{Material} represents the CPU-\/side binding state required before drawing a mesh. It owns\+:
\begin{DoxyItemize}
\item a shared \doxylink{class_shader}{Shader}
\item a set of named uniform values
\item a set of named texture bindings
\item an instancing enabled flag
\end{DoxyItemize}

During rendering, \doxylink{class_render_manager}{Render\+Manager} binds the material, updates view/projection/model and other runtime uniforms, calls \doxylink{class_material_ae4ad039257d5695090655fbd89ef179b}{Send\+Data()}, and then issues the draw call.

The current implementation works as follows\+:
\begin{DoxyItemize}
\item \doxylink{class_material_a1fd6c6c16d1f04e747470a2cc0c52fa1}{Bind()} activates the shader.
\item \doxylink{class_material_ae4ad039257d5695090655fbd89ef179b}{Send\+Data()} sends bound textures first, then sends stored uniform values.
\item \doxylink{class_material_abe3526386e44d6a10fec0e6859dd564a}{Un\+Bind()} unbinds all textures that were bound by this material and then disables the shader. \+:content\+Reference\mbox{[}oaicite\+:3\mbox{]}\{index=3\}
\end{DoxyItemize}

\doxylink{class_material}{Material} also supports instancing when\+:
\begin{DoxyItemize}
\item instancing has been explicitly enabled on the material
\item the shader reports support for instancing
\end{DoxyItemize}

The shader support check is ultimately based on whether the linked shader contains an attribute named "{}i\+\_\+\+Model"{}.

\begin{DoxyNote}{Note}
This class stores rendering state only. It does not perform draw calls directly.

The texture and uniform maps are keyed by uniform name strings, so the caller must ensure the names match the shader source. 
\end{DoxyNote}

void Material\+::\+Enable\+Instancing (\begin{DoxyParamCaption}\item[{bool}]{enable}{, }\item[{std\+::shared\+\_\+ptr$<$ \mbox{\hyperlink{class_mesh}{Mesh}} $>$}]{mesh}{}\end{DoxyParamCaption})}



Enables or disables instanced rendering support for this material. 

When enabling instancing, the current implementation validates\+:
\begin{DoxyItemize}
\item mesh is not nullptr
\item shader is not nullptr
\item shader supports instancing
\end{DoxyItemize}

If all checks pass and enabling is requested, the material marks instancing as enabled and asks the mesh to prepare instance attributes by calling \doxylink{class_mesh_a66fa02b2a66182031bb00a9aec134526}{Mesh\+::\+Setup\+Instance\+Attributes()}.

If the shader does not support the required "{}i\+\_\+\+Model"{} attribute, the request is ignored and a warning is logged.


\begin{DoxyParams}{Parameters}
{\em enable} & True to enable instancing, false to disable it. \\
\hline
{\em mesh} & \doxylink{class_mesh}{Mesh} whose instance vertex layout should be prepared.\\
\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
Disabling instancing only updates the material flag. It does not remove instance buffers already created on the mesh. 
\end{DoxyNote}
