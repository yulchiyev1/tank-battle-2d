\doxysubsection{Detailed Description}
Lightweight context container that provides access to core engine systems. 

\doxylink{struct_engine_context}{Engine\+Context} is a non-\/owning structure used to pass references to core engine systems (such as rendering, input, audio, state management, and window management) throughout the engine and gameplay code.

Instead of relying on global variables or singletons, the engine passes \doxylink{struct_engine_context}{Engine\+Context} to functions such as \doxylink{class_game_state_a6d11e92e629e2d4cbba792faeb75a71f}{Game\+State\+::\+Update()}, Draw(), \doxylink{class_object}{Object} update loops, and resource loading.

All members are raw pointers and are owned by \doxylink{class_jin_engine}{Jin\+Engine}. \doxylink{struct_engine_context}{Engine\+Context} does not manage the lifetime of these systems.

Typical usage includes\+:
\begin{DoxyItemize}
\item Accessing \doxylink{class_render_manager}{Render\+Manager} to submit draw calls
\item Accessing \doxylink{class_input_manager}{Input\+Manager} to query input state
\item Accessing \doxylink{class_state_manager}{State\+Manager} to change game states
\item Accessing \doxylink{class_window_manager}{Window\+Manager} for resolution or viewport information
\item Accessing \doxylink{class_sound_manager}{Sound\+Manager} for audio playback
\end{DoxyItemize}

\begin{DoxyNote}{Note}
All pointers are expected to be valid during runtime after engine initialization.
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
\doxylink{struct_engine_context}{Engine\+Context} does not perform null checks internally. Users must ensure that the engine is properly initialized before accessing its members. 
\end{DoxyWarning}
