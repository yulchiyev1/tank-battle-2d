Wraps an already existing Open\+GL texture object. 

This constructor does not allocate GPU storage or upload any pixel data. It simply stores an existing Open\+GL texture ID together with metadata such as width, height, and channel count so the engine can treat the external texture like a regular \doxylink{class_texture}{Texture} resource.

This is mainly used by internal engine systems that create textures through other Open\+GL paths, such as render targets or externally managed resources.

Loads an image file from disk and creates a GPU texture from it. 

The implementation uses stb\+\_\+image to load the file contents. Vertical flip on load is enabled before reading, so image data is imported in a form that matches the engine\textquotesingle{}s texture coordinate convention.

After loading succeeds, the constructor forwards the pixel buffer and the requested \doxylink{struct_texture_settings}{Texture\+Settings} to \doxylink{class_texture_a5b7154fd70c6f1d7eacc68c04e31ea92}{Generate\+Texture()}, which allocates immutable texture storage, uploads the base image, applies filter and wrap state, and optionally generates mipmaps.

If loading fails, the texture remains with an invalid or empty state and an error is logged.

\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
The loaded CPU-\/side image buffer is released immediately after the GPU texture has been created. 
\end{DoxyNote}

Creates a GPU texture directly from raw pixel data in memory. 

This constructor is used when image bytes are already available in memory, such as asynchronously loaded textures, generated images, font atlases, or procedural content. The provided width, height, and channel count are stored first, then \doxylink{class_texture_a5b7154fd70c6f1d7eacc68c04e31ea92}{Generate\+Texture()} creates the Open\+GL texture and uploads the supplied data.

Supported channel counts in the implementation are 1, 3, and 4, which map to red, RGB, and RGBA texture formats respectively.

void Texture\+::\+Generate\+Texture (\begin{DoxyParamCaption}\item[{const unsigned char \texorpdfstring{$\ast$}{*}}]{data}{, }\item[{const \mbox{\hyperlink{struct_texture_settings}{Texture\+Settings}} \&}]{settings}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [private]}}



Allocates GPU storage and uploads pixel data according to the given settings. 

This internal helper chooses the Open\+GL internal format and upload format from the stored channel count, creates the texture object, allocates immutable storage, uploads the base image if data is present, applies sampler parameters, and optionally generates mipmaps.

When data is nullptr, the implementation still creates the texture and clears its contents so the texture can be used as an empty resource or destination surface.


\begin{DoxyParams}{Parameters}
{\em data} & Pixel buffer to upload. May be nullptr. \\
\hline
{\em settings} & \doxylink{class_texture}{Texture} creation settings applied to the new Open\+GL texture.\\
\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
The implementation temporarily forces GL\+\_\+\+UNPACK\+\_\+\+ALIGNMENT to 1 so tightly packed image data uploads correctly regardless of row alignment. 
\end{DoxyNote}
