\doxysubsection{Detailed Description}
Manages the active game state and performs deferred state transitions for the engine. 

\doxylink{class_state_manager}{State\+Manager} owns the currently active \doxylink{class_game_state}{Game\+State} and an optional pending \doxylink{class_game_state}{Game\+State} that has been requested through \doxylink{class_state_manager_a5337ecd1c035794817b1de05f961d352}{Change\+State()}.

State changes are not applied immediately when \doxylink{class_state_manager_a5337ecd1c035794817b1de05f961d352}{Change\+State()} is called. Instead, the new state is stored in {\ttfamily next\+State} and the actual transition is processed inside \doxylink{class_state_manager_ab4ebd3189a4623e805d95f768b314acb}{Update()}. This deferred design helps avoid replacing the active state in the middle of user logic or rendering code.

During a state transition, \doxylink{class_state_manager}{State\+Manager} performs the following sequence\+:
\begin{DoxyItemize}
\item calls System\+Free() on the old state,
\item calls System\+Unload() on the old state,
\item stops all active sounds through \doxylink{class_sound_manager}{Sound\+Manager},
\item moves the pending state into current\+State,
\item calls System\+Load() on the new state,
\item calls System\+Init() on the new state,
\item updates all cameras in the new state\textquotesingle{}s \doxylink{class_camera_manager}{Camera\+Manager} to match the current window size.
\end{DoxyItemize}

In addition to state switching, this class also drives the main per-\/frame update and draw flow for the current state.

\begin{DoxyNote}{Note}
\doxylink{class_state_manager}{State\+Manager} owns states exclusively through std\+::unique\+\_\+ptr.

This class is intended to be controlled internally by the engine. \doxylink{class_jin_engine}{Jin\+Engine} is declared as a friend so it can access lifecycle functions such as \doxylink{class_state_manager_ab4ebd3189a4623e805d95f768b314acb}{Update()}, \doxylink{class_state_manager_ae161e57265af366c58b0e2b5d936553d}{Draw()}, and \doxylink{class_state_manager_aa8e5c183bf725b7132f7e4bc800deaab}{Free()}. 
\end{DoxyNote}

Processes pending state transitions and updates the active state. 

This function first checks whether a pending state exists in next\+State. If so, it performs a full state transition.

Transition flow\+:
\begin{DoxyItemize}
\item If current\+State exists\+:
\begin{DoxyItemize}
\item call current\+State-\/\texorpdfstring{$>$}{>}System\+Free(engine\+Context)
\item call current\+State-\/\texorpdfstring{$>$}{>}System\+Unload(engine\+Context)
\item stop all sounds through engine\+Context.\+sound\+Manager-\/\texorpdfstring{$>$}{>}Control\+All(\+Stop)
\end{DoxyItemize}
\item move next\+State into current\+State
\item call current\+State-\/\texorpdfstring{$>$}{>}System\+Load(engine\+Context)
\item call current\+State-\/\texorpdfstring{$>$}{>}System\+Init(engine\+Context)
\item resize all cameras owned by the new state to the current window width and height
\end{DoxyItemize}

After transition handling, if current\+State exists, this function calls current\+State-\/\texorpdfstring{$>$}{>}System\+Update(dt, engine\+Context).


\begin{DoxyParams}{Parameters}
{\em dt} & Delta time for the current frame. \\
\hline
{\em engine\+Context} & Shared engine systems and managers used by the state.\\
\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
\doxylink{class_state_manager_a5337ecd1c035794817b1de05f961d352}{Change\+State()} only queues the transition. \doxylink{class_state_manager_ab4ebd3189a4623e805d95f768b314acb}{Update()} is where the actual swap happens.

Sound playback is forcefully stopped when leaving the current state. 
\end{DoxyNote}
